# 产品定位与边界

## 一句话定位

RepoPulse 是一个面向 AI 开发者的个性化开源项目雷达与学习决策日报系统。它每天从开源项目中筛出真正值得学习、借鉴、复刻或转化为应用机会的项目，并根据用户画像解释推荐理由、学习路径、行动建议和潜在风险。

## 核心目标

RepoPulse 不追求成为“另一个 GitHub Trending”。它要回答的是：

> 今天哪些开源项目值得我投入学习时间，为什么值得学，我应该从哪里开始学？

核心目标分为三层：

1. 发现：从 GitHub Trending、Repository Search 和自定义 watchlist 中收集候选项目。
2. 判断：综合热度、工程质量、学习价值、个人目标进行评分。
3. 转化：把结果整理成可直接阅读、收藏和执行的个性化日报。

## 产品化路径

RepoPulse 的产品化优先走“网站入口 + SBTI 测试 + 个性化雷达 / 日报订阅”，不先做完整 SaaS。公开网站负责解释产品、展示样例、承接 SBTI 测试和订阅转化；核心交付仍然是高质量、可行动的个性化报告。

SBTI AI 开发者画像生成模块是 RepoPulse 的 onboarding 和 profile generator。它用轻松的问题把用户偏好转成结构化 profile，再交给现有日报命令读取。SBTI 不采集 GitHub、不分析 README、不排序项目，也不直接改变 ranker。

完整产品化与变现路线见 `docs/11-产品化与变现路线.md`。

## 目标用户

### 主用户：求职与成长中的开发者

典型画像：

- 正在找工作，需要快速掌握有招聘价值的项目和技术趋势。
- 希望通过优秀开源项目学习工程结构、架构设计和实战应用。
- 有明确方向，例如 Java 后端、分布式系统、AI 应用、Agent 工程化、开发者工具。

核心需求：

- 少看噪音，多看值得投入的项目。
- 不只知道项目火，还要知道它能学到什么。
- 快速判断一个项目是否值得读源码、做复刻、写简历项目或转化为产品想法。

### 次用户：产品型开发者 / 独立开发者

典型画像：

- 关注开源项目背后的应用机会。
- 想发现可以改造、组合、商业化或做工具化封装的方向。

核心需求：

- 发现早期上升项目。
- 看出项目解决的问题、潜在用户和可扩展空间。
- 追踪某类技术或产品形态的趋势。

## 用户任务

RepoPulse 应优先服务这些任务：

1. 每天用 5 到 10 分钟了解值得学习的开源项目。
2. 判断某个项目是否适合自己当前职业目标。
3. 获取可执行的学习路径，例如先读 README、再看 examples、最后看 core 模块。
4. 找到可以写进简历、做成 demo 或二次开发的项目灵感。
5. 长期追踪某些技术方向的上升趋势。

## 产品原则

### 1. 学习价值优先于绝对热度

总 star 很高但结构混乱、文档不足、和用户目标无关的项目，不应自动排在前面。

### 2. 推荐必须可解释

每个推荐都必须回答：

- 它是什么？
- 为什么现在值得关注？
- 为什么适合这个用户？
- 应该怎么开始学？
- 有什么风险？

### 3. 先做稳定日报和订阅交付，再做复杂平台

首版产品的核心是稳定生成 Markdown / HTML / 邮件日报，并验证用户是否愿意持续阅读和付费。网站入口可以先做，但只承担介绍、测试、样例和订阅转化；Web dashboard、向量搜索、多用户系统、复杂反馈系统都属于后续阶段。

### 4. 个性化要服务决策，而不是制造黑箱

用户画像用于改善排序和解释，不应让推荐结果变得不可理解。

### 5. AI 是分析器，不是事实来源

事实来自 GitHub 元数据、README、Release、Issue 等证据。AI 负责摘要、归纳、评分解释和阅读路径建议。

## 明确不做

首版不做：

- 社区讨论和社交功能。
- 完整竞品分析平台。
- 多用户权限系统。
- 复杂 Web dashboard。
- 完整 SaaS 工作台和复杂计费后台。
- 自动 fork / 自动生成项目代码。
- 仅按 star 排名的热榜。
- 没有证据的 AI 主观推荐。

后续也应谨慎做：

- 大规模爬虫和非官方高频抓取。
- 不可解释的深度推荐模型。
- 跨太多数据源导致维护成本失控。
- 为了好看而牺牲信息密度的展示。

## 成功标准

RepoPulse 成功不是因为“列出了很多项目”，而是因为：

- 用户每天愿意打开日报。
- 用户能从日报中挑出 1 到 3 个真正值得深入看的项目。
- 推荐理由让用户感觉“这个判断帮我省了时间”。
- 阅读路径足够具体，能直接开始行动。
- 长期运行后能形成个人学习与项目发现档案。
